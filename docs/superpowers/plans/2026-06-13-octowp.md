# OctoWP Implementation Plan

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax for tracking. TDD for pure logic; integration tests use `FakeWhatsApp`.

**Goal:** Build a single-number, safety-first WhatsApp bulk-messaging Electron desktop app that delivers campaigns in resumable batches with anti-ban throttling.

**Architecture:** Electron two-process. Main (Node) hosts Baileys engine + SQLite + throttled campaign worker; Renderer (React) is the UI talking over a narrow typed IPC bridge. All data local.

**Tech Stack:** Electron · electron-vite · React + TypeScript · Tailwind · better-sqlite3 · @whiskeysockets/baileys · xlsx · zod · vitest · electron-builder.

## Phase 1 — Shared types & pure logic (TDD)
Each module: write test → red → implement → green.
- [ ] `shared/types.ts` — Contact, ListDTO, CampaignDTO, RecipientStatus, Settings, RiskPreset, Progress, WAStatus, WhatsAppPort DTOs.
- [ ] `lib/phone.ts` — `normalizePhone(raw, defaultCc) -> E164 | null`. Tests: local `05xx...`→`+905xx`, already-`+`, junk→null, dedupe key.
- [ ] `lib/spintax.ts` — `renderSpintax(text, rng) -> string`. Tests: `{a|b}` picks one, nested, no-braces passthrough, deterministic with seeded rng.
- [ ] `lib/template.ts` — `applyVars(text, vars) -> string`. Tests: `{ad}`→value, missing var→empty/keep policy, multiple vars.
- [ ] `lib/throttle.ts` — pure helpers: `pickDelay(min,max,rng)`, `isWithinActiveHours(date, from, to)`, `dailyCapFor(warmupDay, settings)`, `needsBatchPause(sentInBatch, settings, rng)`. Tests for each.
- [ ] `lib/presets.ts` — `PRESETS.balanced|conservative|aggressive` constant objects matching spec §6 table. Test: shape + default = balanced.

## Phase 3 — WhatsApp engine

- [ ] `wa-engine/fake.ts` — in-memory implementation: configurable `exists` map, records sent messages, can simulate ban (`exists` ok but send throws `BANNED`), emits status.
- [ ] `wa-engine/baileys-adapter.ts` — real Baileys: `useMultiFileAuthState`, QR via `connection.update`, reconnect w/ backoff except `loggedOut`, `onWhatsApp`, `sendMessage`. (Not unit-tested; manual E2E.)

## Phase 4 — Campaign engine (heart, TDD with FakeWhatsApp)
- [ ] `campaign-engine/engine.ts` — given campaign+recipients+settings+port+clock+rng: loop pending → gate (active hours/daily cap/batch pause/warmup) → render (spintax+vars) → `exists` → send → status update + send_log → delay. Pause/resume. Circuit-breaker on BANNED/loggedOut → `halted`. Emits progress.
- [ ] Tests (injected fake clock + rng + FakeWhatsApp):
  - sends to all valid recipients, marks invalid `skipped`.
  - respects daily cap (stops at N, resumes next "day").
  - active-hours gate defers outside window.
  - batch pause triggers after threshold.
  - BANNED → campaign `halted`, no further sends.
  - resume: pre-marked `sent` not re-sent (idempotent).
  - optout phones never sent.
  - progress events fire with correct counts.
